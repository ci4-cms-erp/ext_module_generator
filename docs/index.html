Overview

What is ModuleGenerator?

The ModuleGenerator is a CodeIgniter 4 CLI command designed to create a modular structure for the ci4ms framework. It generates a complete module directory structure with configuration files, controllers, views, and language files, streamlining the development process.

Key Features

  • Creates a standardized module structure with a single command
  • Generates configuration, routes, controllers, views, and language files
  • Supports both English and Turkish language files out of the box
  • Integrates seamlessly with CodeIgniter's CLI for easy execution
  • Follows CodeIgniter 4 conventions and best practices
  • Customizable templates to fit your specific needs
Usage

How to Use ModuleGenerator

To create a new module, run the following command in your CodeIgniter 4 project directory:

php spark make:module ModuleName

If no module name is provided, the CLI will prompt for one. The module name will be automatically capitalized (e.g., user becomes User).

Install via Composer

Add the package to your CodeIgniter 4 project:

compsoer require ci4-cms-erp/ext_module_generator

Example

$ php spark make:module Admin
Module 'Admin' created successfully!

$ php spark make:module Blog
Module 'Blog' created successfully!

This will create modules named Admin and Blog in the modules directory with all necessary files and folders.

Template Files

Generated Template Files

The generator creates several template files with predefined content to jumpstart your development:

Config/ModuleNameConfig.php

Defines module-specific configuration, including CSRF exceptions and filters.

Config/Routes.php

Sets up routes for the module, including index, create, and update routes.

Controllers/ModuleName.php

A controller with methods for index, create, and update actions, including basic validation.

Language Files

English and Turkish language files with welcome messages and other translations.

Each file is generated using a template method in the ModuleGenerator class, ensuring consistency across all your modules.
